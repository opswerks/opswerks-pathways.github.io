Kubernetes Assessment

Before you begin...

  • Ensure with your instructor that your instance has been reset.
  • Make sure you can login to your instance using your account and password (see here for reference).

Tasks

Core Concepts: Namespace, Deployments

  1. Namespace

    Task: Create a new namespace called pathways.

  2. Deploy a Pod.

    Task: Create a pod in the pathways namespace with the following specifications:

    Pod name: nginx-pod

Image: nginx:latest

  3. Create a Deployment

    Task: Create a deployment in the pathways namespace with the following specifications:

    Deployment name: nginx-deployment

Replicas: 3

Image: nginx:latest

  4. Expose the Deployment

    Task: Expose the nginx-deployment as a service with the following specifications:

    Service name: nginx-service

Type: ClusterIP

Port: 80

    Validation: Verify that the namespace, pod, deployment, and service have been created correctly.

Core Concepts: ConfigMap, Secrets

  1. Create a ConfigMap

    Task: Create a ConfigMap in the pathways namespace with the following data:

    ConfigMap name: app-config

    Data: app.properties: "key1=value1\nkey2=value2"

  2. Create a Secret

    Task: Create a Secret in the pathways namespace with the following data:

    Secret name: app-secret

Data: username=admin

Data: password=pathways2024

  3. Deploy a Pod with ConfigMap and Secret

    Task1: Create a pod in the pathways namespace with the following specifications:

    Pod name: app-pod

Image: nginx:latest

    Task2: Mount the ConfigMap app-config as a volume at /etc/config

    Use environment variables from the Secret app-secret:

    ENV_USERNAME from username

ENV_PASSWORD from password

  4. Create a Multi-Container Pod

    Task:Create a pod in the pathways namespace with the following specifications:

    Pod name: multi-pod

First container:

Name: nginx-container

Image: nginx:latest

Second container:

Name: busybox-container

Image: busybox:latest

Command: ["/bin/sh", "-c", "echo Hello from Busybox; sleep 3600"]

    Validations: 

    Ensure that both the ConfigMap and Secret are mounted as volumes in the pod app-pod.

Verify that the ConfigMap, Secret, and pods have been created and are functioning correctly.

Scheduling

  1. Creating Nodes with Labels

    Tasks:

    - Label one of the existing nodes with role=frontend.

- Label another node with role=backend.

  2. Deploying Pods Using Node Selectors

    Tasks:

    - Create a pod that runs on the node labeled role=frontend.

- Create another pod that runs on the node labeled role=backend.

  3. Use Taints and Tolerations

    Tasks:

    Taint a node with the key dedicated, value frontend, and effect NoSchedule.

    Deploy a pod that tolerates the taint applied to the node.

Application Lifecycle Management

  1. Deploy an Application

    Tasks: Create a namespace named opswerks.

    Deploy a simple Nginx application using a Deployment named nginx-deployment in the opswerks namespace with the following specifications:

    2 replicas

Image: nginx:1.18

Expose: port 80

  2. Perform an Update

    Tasks: 

    - Update the nginx-deployment to use the image nginx:1.19.

- Verify that the update is successful and all pods are updated.

  3. Manage Application Configurations

    Tasks: Create a ConfigMap named nginx-config with the following configuration:

    Key: index.html

Value: A custom HTML content (e.g., Hello from OpsWerks! Pathways!)

Update the nginx-deployment to use the nginx-config ConfigMap.

Verify that the Nginx pods are using the configuration from the ConfigMap.

Troubleshoot the pod

  1. Scenario: You are given a pod definition file that describes a simple web application. However, the pod is not running as expected.

    Your task is to diagnose and resolve the issue.

    Tasks: Correct the pod definition file web-app.yaml in your home directory to ensure the pod runs successfully.

    Note: Do not delete the existing pod

    Validation: Ensure the pod is running and working correctly.
