Midpoint Assessment

Main goal of these series of task is to deploy a kubernetes pod running on your own custom image. Your custom image should be built and published in docker hub.

Pre Requisite: To complete this exercise you will need an existing account in docker hub. If you have no account create an account in https://hub.docker.com/

Validation: Calling your app should display "Hi! I am <name>, and this is my app". 

curl <NODE IP>:<NODE PORT>

Task 1: Create a Custom Nginx Docker Image

Instructions:

    Create a Dockerfile using nginx:alpine as the base image.

    Customize the Dockerfile to display "Hi! I am , and this is my app". Build the Docker image and name it my-custom-docker-<name>. Below is a sample index.html file you can use.

Sample index.html


<!DOCTYPE html>
<html>
<head>
        <title>My Custom Nginx </title>
</head>
<body>
        <h1>Hi I'm <name> and this is my app.\</h1>
</body>
</html>

    Push and publish your created docker image to dockerhub. You will need to login and use your docker hub account for it.

Task 2: Deploy a kubernetes pod named my-custom-pod1 running the custom docker image my-custom-docker-<name>

Instructions:

    Create a pod using the custom docker image you created.

Validation:

    Verify that the kubernetes pod is running and using the my-custom-docker-<name> image.

Task 3: Create a Multi-Container Pod with Custom Docker Image and MySQL Image

Instructions:

    Create a pod named my-multi-container-pod that includes two containers:

      The first container named my-custom-container should use the my-custom-docker-name image created earlier.

      The second container named mysql-container should use the mysql:5.7 image.

      Create an EmptyDir volumeMount named container1vm at /usr/logs in the my-custom-container container.

      Create and EmptyDir volumeMount named sqlvolume /var/lib/mysql in the MySQL container.

Validation:

    Verify that the pod my-multi-container-pod is created and running with the specified containers.

    Ensure that the EmptyDir volumes are correctly mounted in both containers.

Task 4: Expose the Deployment as a NodePort Service

Instructions:

    Create a Nodeport service named my-custom-service to expose the my-multi-container-pod port 80 to 30007 port of a node.

Validation:

    Ensure that the service my-custom-service is created.

Task 5: Create a Kubernetes Secret and mount it on your multi-container pod.

Instructions:

    Create a Kubernetes Secret named my-custom-secrets containing dummypassword:p@ssword.

    Expose the secret as an environment variable MYSQL_ROOT_PASSWORD in the mysql container of the multi-container pod.

Validation:

    Check that the Kubernetes Secret my-custom-secrets is created and used in the multi-container pod.

Task 6: Create a Busybox pod to use a Custom PV and Custom PVC.

Instructions:

    Create a pv named my-custom-pv with below variables.

storage: 1Gi
accessModes: ReadWriteOnce
hostPath: "/mnt/data"
storageClassName: manual

    Create a corresponding pvc named my-custom-pvc to be bounded to the pv.

    Create a busybox pod named my-busybox that executes below command and mounts testfile on the the newly created pvc.

["/bin/sh", "-c", "while true; do echo $(date) >> /mnt/data/testfile; sleep 5; done"]

Validation:

     my-busybox should be running and using the pvc created.
