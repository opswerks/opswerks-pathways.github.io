AWS Assessment

Welcome, Site Reliability Engineer!

AWS Practical Lab Exam

You are a Site Reliability Engineer (SRE) newly hired by ProdWerks, a rapidly growing cloud consultancy firm. Your team has just secured a major contract with MacJolli-C Enterprises, a multinational corporation looking to migrate their development environment to AWS.

MacJolli-C Enterprises has recently subscribed to AWS with the intention of migrating their e-commerce platform and associated services to the cloud. However, they lack in-house expertise in AWS technologies and best practices. As a result, they've entrusted your team (YOU) with a critical mission: to design, provision, and configure a comprehensive development environment in AWS for their Silver Team. This environment will serve as the foundation for their cloud migration efforts and future development work. Your task is to leverage AWS services to create a scalable, secure, and efficient infrastructure that aligns with MacJolli-C's business needs while following cloud best practices.

You've been tasked with setting up a dedicated development environment for their Silver team. You have the necessary access to AWS to create this environment, which should mirror the production setup while allowing for experimentation and testing.

Your mission is to leverage your expertise in AWS services and SRE practices to design and implement a new, robust development environment for the Silver team. You'll be responsible for creating the necessary AWS resources from scratch, ensuring they are scalable, secure, and optimized for development workflows. Your focus will be on setting up a solid foundation that enables efficient development processes and adheres to AWS best practices.

You will be presented with a series of tasks that simulate the process of setting up a comprehensive development environment in AWS for MacJolli-C Enterprises' Silver team. Each task will require you to create and configure specific AWS resources, focusing on the core services needed for their e-commerce platform.

Prerequisites

  • You will be using Kode Kloud's AWS Playground for this lab exam.
  • Open AWS KodeKloud Playground (this will open a pop-up window).
  • Log in to KodeKloud using your OpsWerks credentials.
  • Click Launch Now.
  • Click Start Lab.
  • Take note of the provided Console link, Username, Password, and Expiry Time.
  • Open the console link and enter the provided credentials.
  • The lab exam time may be different from KodeKloud's expiry time. Please check with your lab proctor.

Important

  • Use the US East (N. Virginia) region (us-east-1) for ALL regional resources.
  • Failure to use the correct region and/or zones will invalidate your answers.
  • You can use the checkbox next to each task number to mark your completed tasks. However, these checkboxes are only a visual aid to help you track your progress.

Tasks

1. VPCs

1.1 Create a new VPC. (5 points)

Your first task is to set up the foundational network infrastructure for their AWS development environment. Create a new VPC with the name dev, IPv4 CIDR block 10.0.0.0/23, no IPv6 CIDR block, and ensure you apply the required tags.

1.2 Create a subnet within the dev VPC. (5 points)

You then need to establish a subnet within it for web services. Create a subnet named web-a with IPv4 CIDR block 10.0.0.0/24 in the Availability Zone us-east-1a, ensuring you apply the required tags.

1.3 Create a second subnet within the dev VPC. (5 points)

To enhance availability and fault tolerance, create a second subnet within the dev VPC. This subnet should be named web-b with IPv4 CIDR block 10.0.1.0/24 in the Availability Zone us-east-1b.

2. Internet Gateway

2.1 Create an Internet Gateway and attach it to the dev VPC. (5 points)

To provide internet access for the dev VPC, create an Internet Gateway with the name dev-internet-gateway. Attach this Internet Gateway to the dev VPC you created earlier.

3. Route Table

3.1 Create a route table, add a route to the Internet Gateway, and associate the route table with the 2 subnets (5 points)

To enable internet connectivity for the web subnets, create a route table named internet-route within the dev VPC. Add a route to the Internet Gateway you created earlier, and associate this route table with both the web-a and web-b subnets.

3.2 Modify the 2 subnets to enable auto-assign public IP. (5 points)

Now we need to make sure that our web subnets will have IPv4 Public IPs assigned to the instances that will be created in those subnets.

4. Security Group

4.1 Create Security Group (5 points)

After setting up the network infrastructure, create a Security Group named allow-http with a description of Allows HTTP traffic, within the dev VPC. This Security Group should allow all IPV4 inbound HTTP traffic.

Tip

  • Leave the rest of the settings as they are.

4.2 Create 2nd Security Group (5 points)

Make another security group, this time name it allow-https with the description Allows HTTPS traffic, configure it to allow all IPV4 inbound HTTPS traffic.

Tip

  • Leave the rest of the settings as they are.

5. S3 Bucket

5.1 Create an S3 bucket zone. (5 points)

Silver team's e-commerce group headed by Park Soo Min needs to a place store a lot of digital content. Create an S3 bucket for storage with the name kodekloud-ow-pave-[YOURNAME]-[MMDDYYY]-b1.

Note

  • MMDDYYYY = Today's Month Day Year (date based on AWS Cloud Shell - run date command to be sure)
  • YOURNAME = firstname initial and then your lastname in lower case letters.

5.2 Upload file. (5 points)

She also wants you to test the upload to make sure that the storage solution you provided is working. Prepare any PNG image under 5MB from the internet and name it as picture.png and upload to the newly created S3 bucket.

5.3 Create a second S3 bucket. (5 points)

News of the storage solution you provided to Park's team reached her fellow group lead, Juan, and he also wants a storage solution for him and his group. Create another bucket and name it kodekloud-ow-pave-[YOURNAME]-[MMDDYYY]-b2

Note

  • MMDDYYYY = Today's Month Day Year (date based on AWS Cloud Shell - run date command to be sure)
  • YOURNAME = firstname initial and then your lastname in lower case letters.

5.4 Enable versioning on the 2nd S3 bucket you just created. (5 points)

Their files gets updated every now and then so he needs a storage solution that can track file versions.

6. EKS Cluster

6.1 Create an EKS Cluster (15 points)

Dmitri and Olga from the Silver Team will also be working with containers. They need an EKS cluster named dev-playground with Kubernetes version 1.29. Create a role for the cluster with the following specifications: AWS Service set to EKS, Use Case set to EKS-Cluster, and Role Name as eksClusterRole. Ensure the cluster is associated with the dev VPC and uses the web-a and web-b subnets.

Tip

  • Leave the rest of the settings as they are.
  • You can complete other tasks while waiting for the resource to finish creating.
  • Make sure you verify all tasks are complete/ready before submitting

6.2 Configure kubectl (5 points)

Make sure you can connect to the eks server via kubectl using aws cloud console.
