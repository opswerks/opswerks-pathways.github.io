Linux Administration & Ansible - Final Exam

Before you begin...

  • Ensure with your instructor that your instance has been reset
  • Make sure you can login to your instance using your account and password (see here for reference)
  • You will have two instances: <name>-pathways-i1-a.pathways.opswerks.net and <name>-pathways-i1-b.pathways.opswerks.net
  • Note: Your instances will be rebooted after you've taken your exam

Tasks

You must perform the following tasks accordingly: 1. Your Instance a <name>-pathways-i1-a will be your Initial host where you will be creating your Ansible Playbooks. 2. Your Instance b <name>-pathways-i1-b will be where you will apply your Ansible Playbooks

Hints

  • Ensure that your SSH keys are well deployed in between your two hosts. This will save you a lot of time and effort in doing the exercises.
  • You can use your Instance a to apply and test the steps manually. Just be careful you don't destroy it because it's where your Playbooks are located.

Task 1: Package Management

  1. (5 pts) Install the necessary package to install the nmap command
  2. (5 pts) Install the System Tools package group
  3. (5 pts) Remove the package that contains the bc command
  4. (5 pts) Disable the epel yum repository
  5. (5 pts) Update the kernel to the latest version. Ensure that the old kernel version is not removed

Task 2: User and Group Management

  1. (5 pts) Ensure that environment variable SUBJECT with a value of PATHWAYS-LINUX is exported everytime a user logs in
  2. (15 pts) Create user max. Change the user's primary group to redbull and add f1 to its supplementary group. Create the groups if they are not yet existing
  3. (10 pts) Create users alonso and ricciardo and add them to f1 group as supplementary
  4. (15 pts) Set max's minimum number of days before he can change his password to 7 days, and maximum number to 90 days. He also must be warned 2 days prior of the date of his password expiration
  5. (5 pts) Set max user's User ID Info (GECOS) / comment field to "Max Verstappen"
  6. (5 pts) Set the system so that all newly created users by default will be forced to change their password after 120 days
  7. (10 pts) Create the user noentry. Set their uid to 12345 and permanently set their shell to /sbin/nologin so that they will not be able to login to the system
  8. (5 pts) Create a directory named /mnt/drivers and change its group ownership to redbull. Set the directory's permission so that everytime a file is created inside, the file's group ownership will be automatically set to redbull
  9. (5 pts) Configure /mnt/drivers directory so that it is readable, writable, and accessible to members of redbull, but not to any other user
  10. (15 pts) Copy the file /etc/fstab to /var/log/fstab-copy. Configure the permission of /var/log/fstab-copy so that 1) the user max is able to read and write to it, and 2) the user alonso can read and execute it

Task 3: Storage Management

  1. (5 pts) Create an MBR partition in /dev/sdg with a size of 256MB with partition type of Linux
  2. (10 pts) Format /dev/sdg1 to ext4 and mount it to /mnt/data. Ensure that the mount is persistent and will be mounted during system bootup
  3. (10 pts) Create a software RAID group (RAID1) out of /dev/sdc and /dev/sdd. Format the resulting RAID group in xfs and mount it to /mnt/raid1
  4. (20 pts) Partition /dev/sde to two 128MB partitions /dev/sde1 and /dev/sde2. Ensure that both partitions are using the Linux LVM partition type. Create a volume group called vg-sde that contains both partitions. Then, create a 100MB logical volume named lv-sde from vg-sde. Lastly, Format the logical volume to xfs and mount it to /mnt/lvm
  5. (10 pts) Create another 256MB partition for /dev/sde and ensure that it's using the Linux swap partition type. Create a swap filesystem out of the partition. Set its priority to 1 when activating

Task 4: Network Management

  1. (5 pts) Ensure that IP forwarding (IPv4) is enabled
  2. (5 pts) Delete the existing connection using the eth1 interface
  3. (15 pts) Create a new network setting for eth1 based on the specs in /root/network-settings.txt
  4. (10 pts) Allow port 1100 (TCP) in firewalld's public zone

Task 5: Service Management

  1. (5 pts) Configure your system so that SELinux should be Enforcing
  2. (5 pts) Ensure that firewalld service is running and will run upon boot-up. Install the service if it's non-existent
  3. (15 pts) Ensure that httpd service is running and will run upon boot-up. Install the service if it's non-existent. http and secured http must be added in firewalld's public zone
  4. (5 pts) Service sssd must not be running and must not be run upon boot-up
