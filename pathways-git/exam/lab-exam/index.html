Git Assessment

Before you begin...

  • Ensure with your instructor that your instance has been reset
  • Make sure you can login to your instance using your account and password (see here for reference)

Instructions

  • Use git commands to satisfy the tasks below.
  • Make sure that you have the correct spelling and correct case for the files and branch names
  • Note: You will lose access to the server once the exam ends.

Tasks

Task 1: Installing git

  • The instance does not have git installed yet. Install git.

Task 2: Creating and cloning your own remote repository

  • Let's create a bare repository named exam in /opt. This will serve as your remote repository throughout this exam.
  • Clone this repository in your home directory.

Task 3: Create the initial content

  • Create a file named my_file.txt
  • Commit this file with the commit message: "first commit"
    Fix any issues that may arise
  • Push the branch to the remote repository

NOTE

For Task 4 onwards, make sure you start the Task by changing your present working directory to ~/exam, as well as pointing your branch to master branch

Task 4: Simulate fast-forward merge

  • Checkout a new branch named foobar
  • Commit a new file named foobar.txt with the commit message "Adding foobar.txt to foobar branch"
  • Merge the foobar branch to the master branch

Task 5: Simulate non-fastforward merge

In the previous task, no merge commit was created because it was a fast-forward merge.

  • Checkout a new branch named non-fastforward-merge, create a new file named sample-merge.txt and commit it with the message "for non-fastforward-merge"
  • Back in the master branch, add a new line in my_file.txt saying "this is an extra commit" amd commit this change in master with the commit message "extra-commit"
  • Merge the non-fastforward-merge branch to master

Task 6: Simulate a rebase

  • Now create a new branch called rebase-branch and add a file called rebase-file.txt
  • Make two commits to the rebase-file.txt with the respective commit messages:
    "1st commit to rebase-file.txt"
    "2nd commit to rebase-file.txt"
  • Rebase rebase-branch to master

Task 7: Simulate a merge conflcit

  • Checkout a new branch named dummy_text Add the following lines to my_file.txt

Lorem ipsum dolor sit amet,
consectetur adipiscing elit,
sed do eiusmod tempor incididunt ut
labore et dolore magna aliqua.

Commit this to the dummy_text branch with the commit message "Adding Lorem ipsum dolor dummy text"

  • Go back to the master branch and add the following text to my_file.txt

Lorem ipsum dolors sit amet,
consectetur conesectra elit,
sed do eiusmod temper incididunt ut
labore et dolores magna aliqua.

Commit this to the master branch with the commit message "Adding Lorem ipsum dolor dummy text"

  • Merge dummy_text to master, this will result into a merge-conflict. Keep the 2nd and 3rd lines from dummy_text and retain the rest from master. And commit the fixed file to master with the commit message: "Fixed dummy text"

NOTE

Before ending this exam, ensure that you don't have any untracked, modified, staged files in your workspace.
